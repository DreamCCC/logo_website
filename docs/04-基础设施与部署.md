## 2. SSH 访问记录

```bash
ssh root@87.106.89.20
```

### 2.1 当前状态

| 项 | 状态 |
|----|------|
| SSH 公钥 | 已由服务器管理员加入 `root` 用户的 `authorized_keys` |
| 权限 | 管理员确认 `.ssh` 与 `authorized_keys` 权限正确 |
| 登录测试 | 已通过 `ssh root@87.106.89.20` 成功登录 |
| 首次连接指纹 | 已在本机确认并写入 `known_hosts` |

首次连接时终端会提示主机真实性确认：

```text
The authenticity of host '87.106.89.20 (87.106.89.20)' can't be established.
ED25519 key fingerprint is SHA256:I4+3TZevyyXflymarPdwjakE8ZJaL7Oc7dEpF/Jn5KI.
Are you sure you want to continue connecting (yes/no/[fingerprint])?
```

输入 `yes` 后，服务器会被加入本机 `known_hosts`，后续连接不会再重复提示。

### 2.2 登录成功标志

成功登录后终端提示符类似：

```bash
root@ubuntu:~#
```

这表示当前已经进入远程 Ubuntu 服务器的 `root` shell。

### 2.3 后续安全建议

首次登录后建议（实施阶段执行，非本文档执行清单）：

- 创建非 root 部署用户 + sudo  
- `ufw` 仅开放 22、80、443  
- `fail2ban`、自动安全更新  
- 禁用密码登录（仅密钥）  

## 3. 推荐技术栈（待开发确认）

一期可采用常见、易维护组合：

| 层级 | 建议选项 | 说明 |
|------|----------|------|
| 前端 | **Next.js + React** | 与参考代码 React 一致；适合高质量营销页面 |
| UI | Tailwind CSS + framer-motion + lucide-react | 对齐参考实现 |
| i18n | 前端轻量 i18n / 后续可接 next-intl | 英语 + 德语文案与切换 |
| API | **Python FastAPI** | 轻量 API，便于后续 Excel、报价规则和数据处理 |
| 数据库 | **MySQL 8** | 按客户要求初版落库 |
| ORM | SQLAlchemy（后续可接 Alembic） | 与 FastAPI 配套，便于迁移 |
| 认证 | HttpOnly Cookie + JWT | 邮箱密码登录，无邮箱验证 |
| 反向代理 | **Nginx** | TLS 终结、静态资源 |
| 进程管理 | **systemd** 或 PM2 | Node 应用守护 |
| TLS | **Let's Encrypt** (certbot) | 需域名 A 记录指向 `87.106.89.20` |

已确认采用 **Next.js 前端 + Python FastAPI 后端** 的前后端分离方案；GitHub 作为本机与服务器之间的代码中转。

## 4. 部署拓扑（建议）

```
Internet
    │
    ▼
[Nginx :443]
    ├── /          → Next.js frontend :3000
    ├── /api       → FastAPI backend :8000
    └── /uploads   → 用户上传（本地卷或后期对象存储）
         │
         ▼
    [MySQL :3306 localhost]
```

- MySQL **仅监听 127.0.0.1**，不对公网暴露  
- 上传目录定期备份，病毒扫描与类型白名单  

## 5. 环境变量（规划）

| 变量 | 用途 |

## 8. 与需求阶段的对应关系

| 需求 | 基础设施动作 |
|------|----------------|
| 注册登录 | API + MySQL `users` |
