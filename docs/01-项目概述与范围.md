# 01 — 项目概述与范围

## 1. 业务背景

公司面向欧洲市场（及类似 B2B 零售/品牌场景），主营 **发光标识、亚克力/金属字、灯箱、立面 Logo** 等定制物料的设计、生产、物流与安装。

当前希望通过 **对外客户网站** 实现：

1. **品牌与能力展示**：让客户看到案例（References）及 Logo/发光字等 **物料介绍**  
2. **需求收集**：客户填写尺寸、材质、Logo 样式/参考图等信息  
3. **初步价格感知**：页面上可即时展示 **「起价」（例如 xxx 欧起）**，降低跳出率  
4. **正式报价**：由内部人工汇总工厂材料费、运输费、安装费与利润后，**次日通过邮件** 发送正式报价（一期不自动化）

历史上曾讨论「材料费 + 运费 + 利润」的 **全自动算法报价** 与 **邮件自动发送**；根据最新一轮沟通，**一期明确不实现**，网站定位为 **信息采集与客户账户** 入口。

## 3. 目标用户

- **终端客户**：经销商、品牌方、门店业主等，需定制发光 Logo / 标识  
- **内部运营**（一期人工）：接收入库的需求，转工厂与物流，邮件回复报价  

## 4. 阶段划分

### 4.1 一期（当前确认范围）

| 能力 | 状态 |
|------|------|
| 营销首页（案例、物料介绍、流程说明） | ✅ 需要 |
| 用户邮箱注册 / 登录（无邮箱验证） | ✅ 需要 |
| 界面英语 + 德语切换 | ✅ 需要 |
| 用户信息及询价需求写入数据库 | ✅ 需要 |
| 多步骤需求表单（约 7 步 / 面包屑，可略简化） | ✅ 需要 |
| 瞬时展示「xxx 欧起」 | ✅ 需要（规则可简单，非正式报价） |
| 完整动态定价引擎 | ❌ 不做 |
| PDF / 发票生成 | ❌ 不做 |
| 自动向客户发送报价邮件 | ❌ 不做 |
| 管理后台（价格、规则、产品维护） | ❌ 一期不做（数据可先 Excel/人工，表结构预留） |

### 4.2 二期及以后（愿景，非一期交付）

- 管理后台：价格、产品、配送选项、计算规则 **数据库可配置**（避免硬编码，减少改价开发成本）  
- 动态定价：`pricing_rules` + 工厂/物流数据导入  
- 正式报价单 PDF、邮件自动化  
- 与供应商 Excel 的同步或导入工具  

## 5. 成功标准（一期）

1. 客户可在网站上 **注册并登录**  
2. 客户可浏览 **案例与物料说明**，并完成 **多步骤需求提交**  
3. 提交后可在界面看到 **起价提示**，并知悉 **正式报价将于约 1 个工作日内邮件回复**  
4. 所有账户与提交记录 **持久化在 MySQL**（使用提供的初版 Schema）  
5. 站点部署在指定 **Ubuntu VPS** 上，具备基本 HTTPS 与可维护性  

## 6. 已确认的产品决策

| 项 | 决策 |
|----|------|
| 界面语言 | **英语 + 德语** 双语支持；用户可切换（如顶栏语言选择器） |
| 默认语言 | **英语（EN）**；用户可通过切换器改为德语；选择持久化（cookie / localStorage） |
| 注册验证 | **不需要** 邮箱验证；注册成功即可登录 |
| 内容翻译范围 | 一期：UI 文案、表单标签、提示与成功页；案例/图库描述可先做一种语言或双语字段（二期 CMS） |

## 7. 非目标（一期）

- 替代工厂报价系统  
- 实时精确到分的自动报价  
- 支付、订单履约、生产排程  
- 邮箱验证 / 激活链接注册流程  
- 三语及以上或完整 CMS 多语言工作流（仅 EN + DE）
