# 07 — 前端网站介绍与使用说明


## 2. 网站定位

该网站目前是一个 **Logo / Leuchtreklame / 发光标识客户询价门户**。

当前阶段的目标不是自动生成正式报价，而是：

1. 向客户展示公司能力、案例方向和物料类型
2. 引导客户注册账号
3. 收集客户的 Logo / 发光标识需求
4. 给客户展示一个非正式起价，例如 `from EUR 299`
5. 将客户账号和询价需求保存到服务器 MySQL 数据库
6. 后续由人工查看客户需求，并通过邮件发送正式报价

## 3. 当前前端页面功能

### 3.1 首页

路径：

```text
/
```

功能：

- 深色视觉风格首页
- 顶部固定导航栏
- Hero 大图展示
- Logo / Signage 系统介绍，目前展示 4 类产品：不发光 logo、发光 logo、lightbox、侧面安装 logo
- References 案例展示区
- Material guide 物料介绍区
- How it works 流程说明区
- `DE / EN` 语言切换，默认德语
- `Request quote` 询价入口
- `Account` 用户中心入口

说明：

- 当前首页样式参考了前期给出的 React 示例代码风格
- 当前页面不是最终设计稿，也不是像素级复刻竞对网站
- 后续可以继续做视觉还原、内容替换和交互增强

### 3.2 注册页

路径：

```text
/register
```

功能：

- 客户用邮箱和密码注册
- 可填写公司名、联系人、电话
- 注册后直接进入登录状态
- 当前版本 **不需要邮箱验证**

建议测试数据：

```text
Email: test@example.com
Password: Testpass123
Company: Test Company
Contact: Mason
Phone: 123456
```

### 3.3 登录页

路径：

```text
/login
```

功能：

- 已注册用户通过邮箱和密码登录
- 登录成功后可以访问 `/account` 和 `/quote`

### 3.4 询价页

路径：

```text
/quote
```

功能：

- 多步骤询价表单
- 当前步骤包括：
  1. 项目类型
  2. 尺寸和数量
  3. 材质
  4. 照明方式
  5. Logo / 参考图上传
  6. 安装与地址信息（包含室内/室外安装场景，以及不同安装方式）
  7. 确认并提交

提交后系统会：

- 生成询价编号，例如 `Q-20260527-0002`
- 展示非正式起价，例如 `from EUR 299`
- 将询价记录写入数据库 `quotes`
- 将上传文件信息写入数据库 `quote_files`
- 告知客户正式报价会在 1 个工作日内通过邮件发送

注意：

- 当前起价只是参考价，不是正式报价
- 当前不会自动发送邮件
- 当前不会生成 PDF 报价单
- 当前不会自动计算工厂材料费、物流费和安装费

### 3.5 用户中心

路径：

```text
/account
```

功能：

- 查看当前登录用户的询价记录
- 每条记录展示：
  - 询价编号
  - 状态
  - 项目类型
  - 起价
  - 提交时间

当前状态示例：

```text
submitted
```

表示客户已提交，等待人工处理。

## 4. 当前图片说明

当前网站里的图片全部是 **临时占位图**。

图片来源主要是 Unsplash 在线图片，用于在 MVP 阶段快速验证页面布局和视觉效果。

当前图片位置：

```text
frontend/src/app/page.tsx
backend/scripts/seed.py
```

例如前端首页里有：

```ts
const images = {
  hero: "https://images.unsplash.com/...",
  acrylic: "https://images.unsplash.com/...",
  aluminium: "https://images.unsplash.com/...",
  lightbox: "https://images.unsplash.com/...",
};
```

这些图片目前只起到以下作用：

- 占位
- 展示页面视觉方向
- 验证卡片、Hero、案例区布局

正式上线前建议替换为公司自己的真实素材，例如：

- 发光 Logo 案例图
- 门头标识案例图
- 灯箱案例图
- 工厂生产图
- 安装现场图
- 材质细节图
- 客户门店 / 展厅 / 建筑外立面图片

替换方式：

1. 前端静态替换：直接修改 `frontend/src/app/page.tsx` 中的图片 URL
2. 后端 seed 替换：修改 `backend/scripts/seed.py` 中的 `image_url`
3. 后续如开发后台管理页面，可通过后台上传和维护图库

也就是说，目前不能通过网页后台查看：

## 7. 数据库表结构

当前核心表如下：

| 表名 | 用途 |
|------|------|
| `users` | 客户账号、登录邮箱、密码哈希、公司信息 |
| `quotes` | 客户提交的询价需求 |
| `quote_files` | 客户上传的 Logo / 参考图 / PDF 文件元数据 |
| `products` | 前端物料介绍数据 |
| `gallery_items` | 前端案例 / 图库数据 |
| `starting_price_rules` | 起价规则，例如 `from EUR 299` |

### 7.1 `users`

字段：

| 字段 | 说明 |
|------|------|
| `id` | 用户 ID |
| `email` | 登录邮箱 |
| `password_hash` | 密码哈希，非明文 |
| `company_name` | 公司名 |
| `contact_name` | 联系人 |
| `phone` | 电话 |
| `preferred_locale` | 语言偏好，`en` / `de` |

| `updated_at` | 更新时间 |

| 字段 | 说明 |
|------|------|

| `created_at` | 提交时间 |
| `updated_at` | 更新时间 |

## 8. 常用 SQL 查询

### 8.1 查看最新注册用户

```sql
USE logo_portal;

SELECT id, email, company_name, contact_name, phone, preferred_locale, created_at, last_login_at
FROM users
ORDER BY id DESC
LIMIT 20;
```

### 8.2 查看最新询价记录

```sql
USE logo_portal;

SELECT
  q.id,
  q.quote_number,
  q.status,
  q.project_type,
  q.indicative_price,
  q.indicative_price_label,
  q.locale,
  q.created_at,
  u.email,
  u.company_name
FROM quotes q
JOIN users u ON q.user_id = u.id
ORDER BY q.id DESC
LIMIT 20;
```

### 8.3 查看某条询价的完整表单 JSON

```sql
USE logo_portal;

SELECT quote_number, form_payload
FROM quotes
WHERE quote_number = 'Q-20260527-0002';
```

### 8.4 查看上传文件

```sql
USE logo_portal;

SELECT
  q.quote_number,
  f.original_name,
  f.mime_type,
  f.file_size,
  f.file_path,
  f.created_at
FROM quote_files f
JOIN quotes q ON f.quote_id = q.id
ORDER BY f.id DESC
LIMIT 20;
```

### 8.5 查看起价规则

```sql
USE logo_portal;

SELECT id, name, project_type, material, starting_price, currency, active, sort_order
FROM starting_price_rules
ORDER BY sort_order ASC, id ASC;
```

## 10. 当前限制和后续建议

当前限制：

- 没有后台管理页面
- 没有自动发送邮件
- 没有 PDF 报价单
- 没有正式报价算法
- 没有域名和 HTTPS
- 图片是占位图
- 产品和图库数据暂时通过 seed / 数据库维护

后续建议：

1. 替换为公司真实图片和案例
2. 增加后台管理页面，用于查看用户和询价
3. 增加内部通知邮件，客户提交后通知运营人员
4. 增加域名和 HTTPS
5. 增加报价状态流转，例如 `submitted` → `reviewing` → `quoted`
6. 后续再做正式报价算法、PDF 和自动邮件
